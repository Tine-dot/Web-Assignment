Objective
The purpose of this assignment is to assess your understanding of:
 HTML
 CSS
 Version control using Git and GitHub
 Collaborative software development workflows
You will work in groups and collaboratively improve an existing website project while
following a proper GitHub workflow like what is used in professional software
development teams.

Assignment Instructions
1. Form Groups
Students must form groups (not more than 10 people per group).
Each group must assign the following roles:
 Senior Developer (Team Lead)
o Responsible for reviewing and approving pull requests.
o Ensures coding standards are followed.
 Frontend Developers
o Responsible for implementing changes in HTML and CSS.

(Roles can rotate during the assignment if desired.)

2. Fork the Repository
Each group must fork the following repository:
Repository:

https://priyansh42.github.io/IT-Company-Website
Steps:
1. One member of the group should fork the repository on GitHub (ideally the Team
Lead).
2. The forked repository must remain public so that the lecturer can review your
activity.
3. Add all group members as collaborators to the forked repository.

3. Clone the Repository
Each group member must clone the forked repository to their own computer.
Example command:
git clone &lt;your-forked-repository-url&gt;

4. Git Workflow to Follow
You must follow a proper GitHub collaborative workflow.
Step 1: Create Branches
Each developer should create their own feature branch.
Example:
git checkout -b feature-navbar
Step 2: Work on the Project
Developers will edit the HTML and CSS files to improve the website.
Possible improvements may include:
 Improving page layout
 Improving navigation menus
 Enhancing styling with CSS
 Adding new pages
 Improving responsiveness
 Fixing design or structure issues
Step 3: Commit Changes

Developers must make regular commits with meaningful commit messages.
Example:
git commit -m &quot;Improved homepage layout and navigation styling&quot;
Step 4: Push Changes
git push origin feature-navbar
Step 5: Create Pull Requests
Developers must create Pull Requests (PRs) from their branch to the main branch.

5. Pull Request Review Process
The Team Lead in each group will:
 Review pull requests
 Check code quality
 Ensure coding standards are followed
 Approve or request changes before merging
No code should be merged into main/master branch without approval.

6. Coding Standards
Students must follow good coding practices:
1. Write clean readable and maintainable comment
2. Create meaningful branch names
3. Write meaningful commit messages
4. Comment on your PRs to allow others to understand what you are doing

7. Contribution Requirement
All group members must contribute to the repository.
Your individual mark will depend on:
 Number of commits

 Quality of contributions
 Participation in pull requests
 Code reviews
 Overall activity on GitHub
Your contributions will be evaluated using GitHub commit history and pull request
activity.
